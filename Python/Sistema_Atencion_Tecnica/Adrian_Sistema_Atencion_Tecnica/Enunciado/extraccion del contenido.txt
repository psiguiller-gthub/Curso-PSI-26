Comprension y Clases:

El problema plantea un sistema helpdesk:
Los usuarios generan los tickets,
un ticket es asignado automaticamente a un tecnico por el sistema
el tecnico llevara el ticket y cambiara su estado
una vez cerrado el ticket se notifica al usuario

- el usuario dispone de un id
    abre tickets
    recibe notificaciones

- el ticket tiene descripcion, prioridad, id del usuario que lo ha abierto, tecnico asignado
    (lo habre el usuario y lo gestiona el tecnico)
    solo contiene parametros

- tecnico tiene su id,
    actualiza el estado del ticket
    cerrar el ticket

- el sistema tendra una lista de tecnicos disponibles

# nota 1: El enuenciado no menciona cuantos tickets abiertos puede tener un tecnico
# nota 2: no menciona cuantos tecnicos y usuarios hay o puede haber
